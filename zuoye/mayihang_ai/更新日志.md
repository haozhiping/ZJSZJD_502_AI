# 更新日志

## [2026-05-27] 检索优化策略集成

### 新增
- **`query_optimizer.py`** — 查询优化器模块
  - `rewrite_query()` — 查询重写：将模糊查询改写得更具体详细，补充关键词和概念以提升检索精度
  - `generate_step_back_query()` — 查询扩展（后退一步）：生成更宽泛的查询以获取更广泛的背景上下文
  - `decompose_query()` — 子查询分解：将复杂查询拆解为多个简单子问题
  - `evaluate_query()` — 启发式查询评估引擎，无需额外 LLM 调用即可自动选择合适的优化策略
  - `optimize_query()` — 统一入口，执行评估 + 优化

### 修改
- **`rag_service.py`**
  - `chat_with_rag()` 返回值从 `(answer, sources)` 扩展为 `(answer, sources, optimization_info)`
  - 新增 `_search_and_collect()` 统一检索接口
  - 新增 `_merge_deduplicate_results()` 合并去重多个优化查询的检索结果
- **`main.py`**
  - `/api/chat` 接口响应新增 `optimization` 字段，包含策略名称、选择原因、优化详情、优化后的查询列表
- **`templates/index.html`**
  - AI 回复消息新增优化策略徽章（🔄查询重写 / 🔍查询扩展 / 🧩子查询分解）
  - 新增可折叠的「检索优化调试信息」面板，展示策略选择原因、优化后的查询、子查询列表等调试信息

### 策略选择规则

| 触发条件 | 策略 |
|---------|------|
| 多问句或 2+ 并列标记且查询较长 | 子查询分解 |
| 包含技术参数（版本/命令/API 等） | 后退一步 + 查询重写 |
| 短查询(<30字)且包含宽泛词 | 查询重写 |
| 中等长度(<50字) | 查询重写 + 后退一步 |
| 其他情况 | 查询重写 |

### 涉及的参考资料
- `zuoye/资料/rag_01_检索前_分块优化.py` — 分块与关键词提取
- `zuoye/资料/rag_02_检索前_分块全部.py` — 语义分块与完整 RAG 流程
- `zuoye/资料/rag_03_检索前_查询重写.py` — 查询重写
- `zuoye/资料/rag_04_检索前_查询扩展.py` — 查询扩展（后退一步）
- `zuoye/资料/rag_05_检索前_子查询分解.py` — 子查询分解
- `zuoye/资料/rag_06_检索前.py` — 三种策略综合示例

## [2026-05-26] 企业知识大脑 RAG 系统 — 初始版本

Commit: `96351c4` — feat: submit daily homework for mayihang

### 新增
- **`main.py`** — FastAPI 后端入口
  - `GET /` — 提供前端页面
  - `GET /api/docs` — 查询向量库统计信息
  - `POST /api/upload` — 上传 PDF 并向量化入库
  - `POST /api/chat` — RAG 检索问答接口
- **`rag_service.py`** — RAG 业务逻辑层
  - `process_and_store_pdf()` — PDF 文本提取 → 分块 → 向量化 → 存入 ChromaDB
  - `chat_with_rag()` — 检索 Top3 → 构建 Prompt → 调用千问大模型 → 返回答案与溯源
  - `get_db_stats()` — 获取向量库统计
- **`tool.py`** — 工具函数集
  - `extract_text_from_pdf()` — 基于 pdfminer 的 PDF 段落提取
  - `qwen_ef` — 千问 text-embedding-v3 嵌入函数（OpenAI 兼容模式）
  - `build_prompt()` — Prompt 模板填充
  - `load_Dashcope_llm()` — 千问大模型调用
  - `load_llm()` — 通用大模型调用
  - `prompt_templates` — Docker 专家 / 天文专家 Prompt 模板
- **`VectorDB.py`** — ChromaDB 向量数据库封装
  - `VectorDBConnector` 类：支持持久化存储、分批添加文档（Qwen 每批10段 / ZhiPu 每批5段）、向量检索
- **`templates/index.html`** — 前端聊天界面
  - 玻璃态暗色主题、左右布局（侧边栏 + 聊天区）
  - PDF 拖拽/点击上传、消息问答、引用溯源手风琴展开

### 技术栈
| 层次 | 技术 |
|------|------|
| 后端框架 | FastAPI + Uvicorn |
| 向量数据库 | ChromaDB（持久化） |
| 嵌入模型 | 千问 text-embedding-v3 |
| 大语言模型 | 千问 qwen-plus |
| PDF 解析 | pdfminer |
| 前端 | 原生 HTML/CSS/JS |
